## 7. Diagnosis of Current Issue

### Symptom
- User accesses `https://prelaunch.bearlakecamp.com/keystatic`
- CMS UI loads successfully
- No GitHub login prompt appears
- Cannot save pages

### Hypothesis 1: Storage Mode Not Detecting Custom Domain
**Possibility**: 70%

`VERCEL_URL` environment variable may not be set correctly for custom domain.

**Test**:
```javascript
// Add to keystatic.config.ts temporarily
console.log('VERCEL_URL:', process.env.VERCEL_URL);
console.log('isCustomDomain:', !vercelUrl.includes('.vercel.app'));
console.log('Storage mode:', storage.kind);
```

**Expected on prelaunch.bearlakecamp.com**:
```
VERCEL_URL: prelaunch.bearlakecamp.com
isCustomDomain: true
Storage mode: github
```

**If actually seeing**:
```
VERCEL_URL: bearlakecamp-nextjs-<hash>-travis-projects-3a622477.vercel.app
isCustomDomain: false
Storage mode: local
```

Then config is using wrong deployment URL.

### Hypothesis 2: GitHub App Callback URL Not Configured
**Possibility**: 80%

GitHub App `bearlakecamp-cms` may not have `prelaunch.bearlakecamp.com` in allowed callback URLs.

**Verification**:
1. Go to `https://github.com/settings/apps/bearlakecamp-cms`
2. Check "Callback URL" field
3. Verify includes: `https://prelaunch.bearlakecamp.com/api/keystatic/github/oauth/callback`

**If missing**: GitHub OAuth will fail silently (no redirect, no login prompt).

### Hypothesis 3: Environment Variables Not Available in Production
**Possibility**: 30%

Environment variables may be set but not exposed to custom domain deployment.

**Test**: Check Vercel dashboard → Project Settings → Domains → Ensure `prelaunch.bearlakecamp.com` is assigned to THIS project (not a different one).

### Hypothesis 4: Multiple Vercel Projects Serving Same Domain
**Possibility**: 40%

User mentioned "3 Vercel projects exist". If multiple projects claim `prelaunch.bearlakecamp.com`, requests may route to wrong project.

**Test**: List all Vercel projects and their domains.
