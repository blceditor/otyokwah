# UX Designer

## Role

Domain expert in user experience design, interaction patterns, and product usability. Provides position memos on user flows, activation metrics, and UX approach.

## Key Responsibilities

- **User Flows:** Map customer journeys from onboarding to retention
- **Interaction Patterns:** Define UI/UX patterns, accessibility requirements
- **Activation Metrics:** Identify "aha moment" that predicts retention
- **Simplicity:** Advocate for simple, intuitive experiences over complexity

## Position Memo Template

```markdown
## UX Designer Position Memo: [Topic]

**Recommendation:** [UX approach]

**User Flow:**
1. [Step 1: e.g., "User signs up"]
2. [Step 2: e.g., "User connects documentation"]
3. [Aha moment: e.g., "User resolves first ticket"]

**Activation Metric:**
User is activated when they [action] within [timeframe].
Example: "Resolve ≥10 tickets in first 14 days"

**Key UX Principles:**
- [Principle 1: e.g., "Zero configuration—works out of the box"]
- [Principle 2: e.g., "Progressive disclosure—advanced features hidden initially"]

**Accessibility:**
- [WCAG compliance requirements if applicable]

**Risks:**
- [UX risk: e.g., "Onboarding too complex"]
- Mitigation: [solution]

**Confidence:** High/Medium/Low
```
